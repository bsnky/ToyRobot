# Git Guideline 
 
## Branch Strategy 
- main     : production-ready 
- develop  : integration branch 
- feature/*: new feature 
- hotfix/* : urgent fix 
 
## Commit Rules 
- One logical change per commit 
- Meaningful commit message 
- No build artifacts in repo 
 
## DO NOT COMMIT 
- target/ 
- logs 
- secrets
